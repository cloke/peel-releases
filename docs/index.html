Your tools. Your models.

Peel runs a local server that speaks two protocols, so the coding agents you already use gain Peel's capabilities without changing your workflow.

Tools out — MCP

Peel exposes its tool surface over MCP. Point Claude Code, Claude Desktop, Cursor, or Continue at the local endpoint and Peel's repository, git, and agent tooling appears in the client you already use.

Models out — OpenAI-compatible

Peel also serves a routed model pool over an OpenAI-compatible API. Any OpenAI-SDK client can use Peel's local and routed models through one base URL — including automatic routing.

Setup for each client is documented in the repository. A dedicated in-app Integrations screen is in progress.

What is a swarm?

A swarm is a group of Macs running Peel together. Each Mac becomes a worker that advertises its repositories, hardware, models, and availability. Peel sends a task to the machine best equipped to perform it, shares reusable repository analysis, or routes local-model inference to a stronger GPU.

You can use Peel entirely on one Mac. A local LAN swarm does not require an account. Read the swarm guide →
